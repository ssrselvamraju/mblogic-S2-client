Help - Platform Data

Overview

The "Platform Data" page allows the discovery and monitoring of server platform data. The platform data is information about the computer which the system is running on. This platform data can be helpful in maintaining systems after they have been installed.

The "Platform Data" page is accessed via a link on the "System Monitor" page.

Basic Platform Data

The basic platform data includes the following:

  • Server Host Addr - This is the host (server) name or address.
  • Node - This is the name which was assigned to the computer by the person installing or configuring it.
  • Operating System - This is the operating system running on the server.
  • Platform - This is a more detailed description of the operating system on the server.
  • Python Version - This is the version of the Python language system used to run the system software. This provides supporting services necessary for running the system.
  • Python Revision - This is additional Python language version information.

Extended Platform Data

In addition to the basic platform information, some platforms offer additional ("extended") information. Not all platforms support the ability to collect this information, and on those which do not, the additional table rows will be hidden. The extended platform data includes the following:

  • Total RAM - This is the total amount of RAM installed.
  • CPU Model - This is the model number of the server CPU.
  • Disk Size - This is the size of the disk or partition where home is mounted on. The size is reported in gigabytes.
  • Disk Free - This is the amount of free space on the disk or partition where home is mounted on. The size is reported in gigabytes.
  • Avg Total % CPU Load - This is the total CPU load including all processes running on the computer.
  • Server % CPU Load - This is the total CPU load including only the basic automation platform processes running on the computer. This also excludes generic clients.

Ideally, the "average total CPU load" should report a number somewhat higher than "server CPU load" as it includes additiona loads. However, due to limitations in the operating system's process accounting calculations these two numbers will often disagree, especially at low system loads. Indeed, the "average total" may display a slightly lower value than "server CPU load". These minor discrepencies are unavoidable and the same phenomenon is also visible in "top" (the standard operating system utility which can display the same data in a more detailed form). The CPU load values are intended to be used as a general guide for estimating the amount of work the server is capable of handling, and indeed the numbers reported are more than accurate enough for that purpose.
